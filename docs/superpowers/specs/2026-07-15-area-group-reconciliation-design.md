# EMS Scout 区域组规则对账设计

## 1. 目标与边界

“区域组与关注”收敛为“区域组”。删除面向用户的关注设备、关注时间窗、关注事件和相关数据筛选；相关 UI、依赖注入和评估链路退役。SQLite 中既有 `device_watch_rules` 表与数据继续作为兼容状态受保护，导入完整性指纹仍会读取它，所属区域组被显式删除时会在同一事务中清理关联旧记录。

区域组是用户可维护的容器。所有区域组均可改名、停用和删除。系统首次升级到 V6 时预设“公区”和“非公区”两个普通区域组及对应分类规则；预设只执行一次，用户删除后不得自动重建。

## 2. 核心模型

V6 新增四类数据，继续使用 `monitor_groups` 作为区域组主表：

1. `area_group_rules`：持续匹配规则。支持公区分类、非公区分类、整层覆盖、设备名精确匹配、设备名关键字和只读兼容子区规则。
2. `area_group_members`：已经确认的设备成员。成员来源为规则确认、手动加入或旧数据迁移。
3. `area_group_exceptions`：按区域组和物理设备隔离的长期例外。`blocked` 表示长期屏蔽加入，`retained` 表示规则不再命中时手动保留。
4. `area_group_change_requests`：规则对账产生的加入或移除候选及其处理历史。

设备身份优先使用 `device_uid`。旧数据没有 UID 时，使用楼栋、楼层、子区、设备名、来源键和出现序号组成稳定回退键。一个区域组内同一物理设备只能有一条正式成员、一条例外和一个同动作的待处理请求。

## 3. 成员与规则语义

- 用户从现有设备目录选择单台设备时，操作本身就是人工确认，直接建立 `manual` 成员。
- 用户选择整层时，保存一条持续 `floor` 规则；当前及以后采集到的该楼层设备均先进入待确认加入。
- 用户确认完整设备名时，保存持续 `name_exact` 规则；现在及以后同名设备均先进入待确认加入。
- 用户保存设备名关键字时，保存持续 `name_keyword` 规则；现在及以后匹配的设备均先进入待确认加入。
- 不允许新建整栋规则；整层是最粗的人工选择范围，单台设备是最细范围。
- 旧 `sub_area` 成员迁移为只读兼容规则，保证已有数据不丢失，但新 UI 不再提供新建子区规则。

## 4. 对账状态机

每次成功采集导入，在当前卡片和楼层目录写入完成后、同一 SQLite 事务提交前执行对账。

1. 计算本次导入楼栋中每个启用区域组的规则命中设备。
2. 命中但不是正式成员、且未被 `blocked` 的设备产生 `pending/add`。
3. 已确认的 `rule` 成员不再命中、且未被 `retained` 时产生 `pending/remove`。
4. `manual` 成员不因规则变化自动移除。
5. 已不适用的待处理请求标记为 `superseded`，不直接删除历史。
6. 精确重放在导入器现有幂等返回点结束，不重复对账。
7. 单栋/部分导入只计算被导入楼栋，绝不据此移除其他楼栋成员。

优先级固定为：长期屏蔽阻止待加入；手动保留阻止待移除；人工成员不受规则移除；否则以启用规则集合为准。

## 5. 审计动作

区域组页面只显示“待确认加入/待确认移除”数量和跳转审计入口。审计页面承担正式动作：

- 确认加入：建立或恢复 `rule` 成员，请求记为 `accepted`。
- 拒绝加入：建立/更新 `blocked` 例外，可填写备注，请求记为 `rejected`。
- 确认移除：删除对应 `rule` 成员，请求记为 `accepted`。
- 拒绝移除：建立/更新 `retained` 例外并保留成员，可填写备注，请求记为 `rejected`。
- 解除屏蔽或取消保留：删除例外；下一次对账按当前规则重新产生候选。

所有动作必须校验请求仍处于 `pending`，并在一个事务内更新请求、成员和例外。

## 6. 页面设计

### 区域组

页面保留左侧区域组列表和右侧工作区。右侧按顺序显示：

1. 区域组资料与启用状态；
2. 待确认加入/移除提醒及“打开审计”；
3. 持续规则：预设分类、楼层、设备名精确匹配和设备名关键字，可编辑、停用或删除；
4. 现有设备目录：楼栋、楼层、设备，支持搜索设备名；
5. 正式成员：来源、设备位置、可编辑备注和移除；
6. 例外名单：长期屏蔽/手动保留、备注、解除。

原“楼层候选目录”和“关注设备”卡片删除。“现有设备目录”只读取当前 `cards` 数据，不允许在这里创造不存在的设备。

### 审计

审计页面新增“分组成员变更”区域，支持按区域组和动作筛选。每行展示区域组、设备、位置、命中依据、检测批次和当前动作，并提供确认、拒绝及备注输入。区域组页面的提醒链接带上区域组筛选。

## 7. 数据安全与兼容

- V6 仅使用 `CREATE TABLE/INDEX`、`INSERT`、`UPDATE`，不删除旧表和旧列。
- 旧 `monitor_group_items` 保留，迁移只复制到新模型；旧时间计划表继续可读但不恢复已退休页面。
- `device_watch_rules` 数据保留，应用不再注册或调用关注服务。
- 导入保护指纹新增规则、成员和例外表；候选表允许在导入事务中变化。
- 所有测试和现场验证使用临时数据库，不改写 `data/ac.db` 或 `out/ac.db`。

## 8. 错误处理

- 规则参数无效、整栋规则、空关键字、跨组请求处理和重复处理均抛出明确异常。
- 对账失败回滚整个导入事务，不留下部分卡片或部分候选。
- UI 刷新失败保留现有列表并显示状态文本；确认动作失败不提前移除行。

## 9. 验收
