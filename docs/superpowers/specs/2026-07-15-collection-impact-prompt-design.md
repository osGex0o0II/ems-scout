# 采集范围影响提示三态设计

## 根因

`CurrentDataImpactText` 当前只根据已选楼栋数量是否大于零生成文案，因此全选六栋时仍显示“其他楼栋保持不变”。这段文案在采集页左侧卡片的有限宽度下还会产生额外换行，使全选状态的卡片高度与用户预期不一致。

## 设计

采集范围提示按选择状态分为三态：

1. 未选择楼栋：显示“选择至少一栋楼后才能开始”。
2. 选择部分楼栋：显示“成功后只更新……，其他楼栋保持不变”。
3. 选择全部楼栋：隐藏影响提示控件，整段提示不占布局高度。

判断以 `Buildings.Count` 与已选数量比较为准，避免依赖硬编码的楼栋数量。提示内容仍由 ViewModel 生成，XAML 只绑定文案与可见性。

全选时同时将楼栋摘要压缩为“已选择全部 N 栋楼”，不再展开完整楼栋列表；这样可以避免标题区被右侧“全选/清空”按钮挤压后出现孤立分隔符或漏掉末尾楼栋。

## 验证

- UI 合同测试覆盖三态文案和全选隐藏绑定。
- 原生构建与采集页合同测试通过。
- 实际页面验证未选择、部分选择、全选三种状态的文案和卡片布局。
- 全选摘要在有限宽度下保持语义完整，不裁切楼栋编号。

## 非目标

- 不改变楼栋选择命令、采集执行计划或数据库更新范围。
- 不调整采集页其他卡片和运行记录功能。
