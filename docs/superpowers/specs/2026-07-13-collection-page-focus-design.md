# EMS Scout 采集页聚焦设计

## 目标

把采集页收敛为普通用户可以直接理解的主流程：选择楼栋，确认采集浏览器和 EMS 登录就绪，然后开始采集。保留现场补采和完整采集能力，但不再把内部流水线步骤当成普通任务选项展示。

## 设计原则

- 默认路径只展示完成任务所需的信息，不要求用户理解 Node、Sidecar、CDP、JSON 或 SQLite。
- 高频选项留在采集页，低频工程参数不与日常采集混排。
- 运行前检查先回答“能不能开始”和“卡在哪里”，技术明细按需展开。
- 采集浏览器由应用明确拥有，打开和关闭使用同一个状态按钮；采集中禁止关闭。
- 不修改采集规则、通讯状态映射、数据库结构或生产数据路径。

## 页面结构

### 采集范围

- 六个楼栋统一显示为 `1号楼`、`2号楼`、`3号楼`、`4号楼`、`5号楼`、`6号楼`。
- 内部传给枚举器的值继续使用 `1号` 至 `6号`，只改显示文本。
- 在缺少六栋正式全称的权威来源时，不单独给 1 号楼显示“科研综合楼”。

### 运行前检查

- 顶部状态条保留，但标题改为明确结果：`正在检查`、`运行前检查通过`、`运行前检查未通过`。
- 未通过时，摘要只显示当前最先需要用户处理的卡点和动作，例如：`卡在采集浏览器：请先打开采集浏览器` 或 `卡在 EMS 登录：请在浏览器中完成登录后重新检查`。
- 通过时显示：`采集浏览器、EMS 页面和本地采集组件均已就绪`。
- 四项检查始终显示在紧凑状态面板中，不再折叠；面板顶部显示通过比例和进度条，例如 `2/4 已通过`。
- 每项使用浅灰圆形底和低饱和图标区分状态：通过为灰绿色、阻塞为灰棕色、待检为中性灰；状态文字保持主题灰色。重新检查时使用轻量错峰进入和重排动画提示状态刷新，不使用持续闪烁。
- 明细按用户概念合并为：本地采集组件、当前数据、采集浏览器、EMS 页面。Node 运行时、依赖、Sidecar 和原生数据流程归入“本地采集组件”的详情，不再占据四行。
- 页面加载、打开浏览器、关闭浏览器后自动检查；保留 `重新检查` 按钮。

## 高级设置

- `高级任务` 改名为 `高级设置`，副标题为 `采集方式、补采和诊断选项`。
- 普通用户只看到三种采集方式：
  - `标准采集`：枚举、校验、导入和基础质量检查，作为默认值。
  - `完整采集`：标准采集后继续采集实时详情并执行实时审计。
  - `补采指定页面`：显示补采定位输入，完成后仍执行校验、导入和基础质量检查。
- `仅校验 JSON`、`仅导入 SQLite`、`仅基础审计`、`仅实时详情`、`仅实时审计` 不再出现在采集页任务下拉框中。底层执行计划保留，供诊断工具、自动化或后续维护入口使用。
- 补采定位只在选择 `补采指定页面` 时显示，并给出格式示例。
- 保存诊断日志保留为一个开关。日志类别、批量数、超时、重开间隔、最大设备数、跳过清单检查、自检和网络监听等工程参数不再与普通采集方式混排；本轮从采集页隐藏并继续使用安全默认值。

## 采集浏览器生命周期

- 系统设置移除 `默认采集模式`，不再提供 `自动启动 Edge`。
- 旧设置中的 `auto-launch` 在加载时兼容归一化为 `edge-cdp`，避免升级后保留隐式自动启动行为。
- 采集页按钮状态：
  - 未打开：`打开采集浏览器`。
  - 应用拥有的浏览器正在运行：`关闭采集浏览器`。
  - 采集中：按钮禁用，保持浏览器运行，提示 `采集期间不能关闭浏览器`。
- 关闭操作只终止 EMS Scout 本次启动的隔离 Edge 和对应临时 profile，不关闭用户自己的 Edge。
- 打开或关闭后立即刷新运行前检查。浏览器被用户从外部关闭时，下次检查恢复为未打开状态。
- 任务开始不再自动启动浏览器；没有可采集 EMS 页面时明确阻止启动。

## 状态与错误处理

- 运行前卡点按顺序选择第一项：本地采集组件、有效 EMS 地址、采集浏览器、EMS 页面或登录态、当前快照或数据库（仅相关任务需要时）。
- 检查异常时保留可操作的失败摘要，技术异常写入运行记录。
- 关闭浏览器失败时不伪装为已关闭；状态按钮和检查结果以实际进程状态为准。
- 采集结束、失败或取消后重新允许关闭浏览器，浏览器不自动关闭，便于现场复核。

## 验证

- 单元测试覆盖旧 `auto-launch` 设置的归一化、三种用户采集方式对应的执行计划、运行前摘要的首个卡点和通过计数。
- UI 合同测试覆盖统一楼栋标签、始终可见且带状态动画的检查面板、三种采集方式、设置页移除自动启动选项、打开/关闭按钮绑定和采集中禁用。
- 构建和完整原生测试通过后，使用隔离 UI 验证配置检查窄窗口与常规窗口，不读取或修改生产数据库。

## 非目标

- 不改变枚举器、导入器、质量规则和实时采集算法。
- 不为尚未确认的楼栋全称补造名称。
- 不删除底层维护任务模式或 CLI 能力。
- 不在本轮重新设计任务进度、历史批次和运行日志区域。
