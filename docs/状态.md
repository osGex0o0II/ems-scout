## 目标
- 历史目标：完善 EMS 空调数据采集脚本（enumerate.js）并生成准确的 6 栋楼 Markdown 设备清单

## 偏好
- 历史端到端流程：枚举 → 入库 → 生成报告
- 设备名用 SVG 文本提取（稳定），字段值从 Vue 状态富化
- Vue 富化无效值自动回退到 SVG
- 用户偏好：先报告问题再实施，避免无效等待
- 报告层修改优先于重采（无副作用）

## 关键约定
- "0-0001-KT" = **数据未加载完成的占位符**（非真实设备名）
- 1号 2F/3F 裙楼/塔楼分组依据：`QL-` 前缀 + sub-page name
- 5号/6号楼层标题直接用 `sa.text` 原文（"BM"、"1F"），不用 `Fmt-Floor`
- **BM 排最前**（pageOrder `BM=-1`）
- **节点箱（如 "5D-DXS-KT"）在报表层去重**（实际未实现，保持原样）
- 设备命名：1号=房间号，2号="F-NN"（含 1.5F/2.5F），3号/4号="NN-NNNN"，5号="5X-DXX-KT"（5D-DXS-KT=节点箱），6号="XF-NNN-KT"
