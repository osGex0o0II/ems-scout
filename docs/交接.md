## 关键算法

### Get-Zuo5 (5号座号 x 边界)
```powershell
A ≤ 400, B ≤ 720, C ≤ 920, D ≤ 1120, E ≤ 1400, F = 其他
```

### Get-Zuo (6号座号 x 边界)
```powershell
A ≤ 650, B ≤ 1220, C = 其他
```

### pageOrder (楼层排序)
```powershell
"BM" = -1     # BM 排最前
"-2" = -2     # B2F
"0"  = 0      # 数据加载失败兜底
"1"  = 1, "1.5" = 1.5, "2" = 2, ...
```

### 楼层文本
- 1号/2号/3号/4号：`Fmt-Floor` 输出（如"2F"、"1.5F"）
- 5号/6号：sa.text 原文（如"BM"、"1F"）

### 1号 2F/3F 裙楼/塔楼分组
- 依据 sub-page name 包含"裙楼"/"塔楼" + 设备名前缀（`QL-` vs 其他）
- 仅 1号 2F/3F 触发

## 已知遗留

### Bug #1: 5号节点箱跨座显示
- 现象：`5D-DXS-KT` / `5E-DXS-KT` 出现在 D/E/F 座
- 根因：5号节点箱本身跨多子区（如 `5D` 子区含 D-E-F 三座）
- 决策：**报表层不去重**（用户选择保持原样）

### Bug #2: 6号子区重复（群控/公区）
- 现象：6号 C座 1F 在 y=155 和 y=218 各有同 24 卡
- 根因：群控层 y=155 + 公区层 y=218 数据同步
- 决策：**保持现状**（用户接受 cosmetic duplicate）

### Bug #3: recapture 模式不遍历子 tab
- 现象：单子区 recapture 采到 1 页就停（5/6号有"一页/二页/群控/公区"）
- 根因：recapture 主流程未调用 `findSubTabs`
- 影响：仅 6号 BM 18卡已手动补采

### Bug #4: 6号子区重复率 43.6%
- 32 子区中 ~14 个 y=155 与其他 y 重复数据
- 根因未深入排查

### Bug #5: enumerate.js 子页名带前缀
- JSON 内 sub-page name 含 `塔楼/` / `裙楼/` 前缀
- dump 层已处理（去重），JSON 数据未清理

### Bug #6: 中文 PowerShell 乱码
- macOS 编码环境下 PS 输出中文常乱码
- 已规避：直接 Read 工具查看文件

### Bug #7: 0-0001-KT 占位符
- 现象：WS 未推送时设备名为 "0-0001-KT"
- 修复：枚举器 `waitForLoadedCards` + 入库过滤双重保险

## 历史 Bug 已修复

- `clickById` 缺 `clientX/Y` → 设置元素中心
- 楼层导航不刷新子区 → 每层重新扫描

- extractCards grid 漏 row 2 → 阈值 8→2
- findSubTabs 漏 HTML 群控/公区 → querySelectorAll
- pageOrder BM 排最后 → BM=-1
- saveOutput 只存最后楼栋 → 累加所有楼栋
- saveOutput APPEND=false 重置数据 → 始终 load existing
