# EMS Scout 交接

更新时间：2026-07-11

## 从这里开始

1. 阅读 `docs/architecture.md` 了解语言、进程和数据所有权。
2. 阅读 `docs/状态.md` 了解当前基线和未完成门槛。
3. 阅读 `docs/项目规范.md` 了解命名、数据、代码、测试和 Git 规则。
4. Windows 接续验证按 `docs/Windows验证清单.md` 顺序执行。
5. 删除旧路径前必须查阅 `docs/legacy-inventory.md`。
6. 契约变更以 `contracts/` 和对应 fixture/test 为准。

## 本地可执行验证

```bash
node --test sidecar/test/*.test.js tests/architecture/*.test.js tests/contract-audit/*.test.js tests/enumeration/*.test.js tests/field-e2e/*.test.js tests/golden/*.test.js
npm run self-test

npm run native:test

dotnet format native/EmsScout.Native.slnx \
  --verify-no-changes --no-restore
```

macOS 可以验证 Domain/Application/Infrastructure 和测试项目，但不能执行 Windows
`XamlCompiler.exe`。Desktop 构建在 XAML 阶段失败不等同于 C# 逻辑失败，也不能当作
Windows 编译通过。

`npm run native:test` 是干净克隆门禁，排除本地没有的 `Fixture=ProductionEvidence`。
安全放置完整 `out/` 证据后，可用 `npm run native:test:evidence` 单独验证该组。

## Windows 验证

新设备先按 `docs/Windows验证清单.md` 完成环境、克隆、恢复和干净克隆门禁。

```powershell
npm run native:build
npm run native:test
npm run native:run

powershell -NoProfile -ExecutionPolicy Bypass -File scripts\prepare-sidecar.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\package-native.ps1

powershell -NoProfile -ExecutionPolicy Bypass -File scripts\field-e2e.ps1 `
  -Building 1号 -LaunchEdge -RunSingleBuilding
```

现场命令必须保持唯一 runDir、随机 loopback CDP、独立 Edge profile、默认清理和生产
DB 路径拒绝规则。

## 生产数据保护

- 不通过 SQLite 打开 `out/ac.db` 做测试。
- 需要 run17 内容时，先用普通文件复制到临时目录，再对副本运行 SQLite。
- 不删除或改写 `data/1号楼`、`data/2号楼` 下的 WAL/SHM 证据。
- 任何验证前后都应核对生产 JSON/DB/WAL/SHM 哈希。
- 测试输出只能进入系统临时目录或唯一 `out/field-e2e-*` 目录。

## 产品入口

- 采集：原生应用“采集任务”。
- 浏览和筛选：原生应用“数据管理”。
- 导出：`数据管理 -> 导出当前筛选 Excel`。
- 质量与对账：原生应用“审计中心”。
- 日志：原生应用“诊断”，原生日志位于
  `%LOCALAPPDATA%\EMS Scout\logs\native-YYYY-MM-DD.ndjson`。

旧 Web、Electron、TUI、Node import、Node 基础质量和 legacy 报表不是产品主入口。

## 需要保留的事实

- `IND_MAP` 必须位于 Vue try/catch 外部。
- indicator 映射：绿色关机、红色开机、灰色离线；缺失不允许多数派猜测。
- `0-0001-KT` 是未加载占位符。
- `2.5F`、`B1F`、`BM` 都是合法楼层标签。
- 2号当前唯一卡片基线是 107；旧 110 包含 2.5F 同页重复卡。
- 质量标签不能覆盖内容级重新审计。
- 导入、迁移、取消和历史恢复必须保持事务与用户数据完整性。
