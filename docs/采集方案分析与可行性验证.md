# EMS 空调数据采集 — 效率提升方案分析与可行性验证

> 分析日期：2026-06-06

## 1. 当前方案回顾

`ems-tool/src/enumerate.js`（705 行）通过 Playwright + Edge CDP 模拟 UI 点击采集数据：

```
菜单 → 楼栋 → 楼层 → 子区 → 分页 → SVG DOM 解析提取卡片
```

**瓶颈**：
- 每页提取依赖 SVG `<text>` 元素位置匹配 + `<image>` href 对比，代码 ~200 行，脆弱
- 单次 floor 点击等待 2s，全量 6 栋楼约 15-20 分钟
- 已有 7 个位置/正则相关 bug（5号楼层标签误认、裙楼sub-tab漏检、半楼层漏采等）

---

## 2. 方案一：直连 WebSocket（跳过浏览器）

### 思路

不启动浏览器，直接用 Node.js/Python 连接 EMS WebSocket，订阅数据帧。

### 可行性结论：暂不可行

| 缺失项 | 详情 |
|--------|------|
| WebSocket URL | 从未捕获到。`events.jsonl` 只有登录页事件，`Network.webSocketCreated` 未记录 |
| 通信协议 | 推测 STOMP over SockJS，但 `chunk-vendors.js` 中未找到明确证据 |
| 认证机制 | JWT token 获取路径不明，登录流程需完整逆向 |
| 订阅消息格式 | 厂商 SDK（`front-basic-components`）封装了全部 WS 通信 |

**投入产出比低**，不建议现在投入。

---

## 3. 方案二：Vue 状态直读替代 SVG 解析 ★推荐

### 思路

保持 Playwright 导航（点击菜单/楼层），但**不再解析 SVG DOM**，改为读取 Vue 组件内存状态：

```javascript
// 当前方案（~200行，脆弱）
const texts = svg.querySelectorAll('text');
const images = svg.querySelectorAll('image');
// ... 位置匹配、正则过滤、开关图标对比 ...

// 新方案（~30行，结构化）
const data = await page.evaluate(() => {
  const w = document.querySelector('.pi-graphics-configuration-svg-new');
  const d = w.__vue__.$data;
  const rc = d.runConfDataProp;   // 测点配置
  const ws = d.websocketDataProp;  // 实时值
  
  const byDev = {};
  for (let i = 0; i < rc.length; i++) {
    const c = rc[i].ptPathConf;
    if (!c.devId) continue;
    if (!byDev[c.devId]) byDev[c.devId] = { cards: [] };
    byDev[c.devId].cards.push({
      idx: i,
      name: c.name,           // "室内温度", "系统模式设置" 等
      dynType: c.dynType,     // 1=开关, 4=模式, 9=温度, 11=设备名, 23=风速
      value: ws[String(i)]?.tag.value,
      enumDef: c.enumDefine   // 枚举映射
    });
  }
  return byDev;
});
```

### 可行性结论：完全可行

#### 证据链

**a) 设备名在 Vue 状态中**

来自 `ems-mirror/api/GetDevAttr.json`：
```json
{"ptPathKey":"4c83553e9689a8bcb1a9e2af741056c3","tag":{"value":"XXFDT-KT"}}
{"ptPathKey":"6e7713ef254563f38f69e2041c7dca4e","tag":{"value":"DXFDT-KT"}}
{"ptPathKey":"bbab830b8edf42001cb5639855a743e5","tag":{"value":"105-KT-1"}}
```

设备名是 `dynType=11` 的点的 `tag.value`。

**b) 5 字段完整映射**

| 字段 | dynType | ptPathConf.name | 枚举映射 | 来源 |
|------|---------|-----------------|----------|------|
| 设备名 | 11 | null（存为 value） | — | `ws[i].tag.value` |
| 开关 | 1 | "当前开关机模式" | {0:"关机",1:"开机"} | `ws[i].tag.value` |
| 模式 | 4 | "系统模式设置" | {0:"通风",1:"制冷",2:"制热"} | `ws[i].tag.value` |
| 室内温度 | 9 | "室内温度" | — | `ws[i].tag.value` |
| 设定温度 | 9 | "当前设置温度" | — | `ws[i].tag.value` |
| 风速 | 23 | "设定风速" | {1:"低",2:"中",3:"高",4:"自动"} | `ws[i].tag.value` |

**c) 设备分组**

每个 `devId` 对应 7~9 个测点，按 `devId` GROUP BY 即可得到每台设备的完整状态。

**d) 已有脚本验证**

`ems/ac_table.ps1`（33-49 行）已在用 `dynType` 过滤点：
```powershell
$sw   = & $get 1    # 开关
$mode = & $get 4    # 模式
$temp = & $get 9    # 温度
$fan  = & $get 23   # 风速
```

`ems/realtime_wait.ps1` 已处理 WS 等待逻辑（`Start-Sleep -Seconds 5`）。

#### 预估效果

| 指标 | 当前 SVG 解析 | Vue 状态直读 |
|------|-------------|------------|
| 单页提取耗时 | ~1-2 秒 | ~50-100ms |
| 提取代码行数 | ~200 行 | ~30 行 |
| 依赖项 | SVG 结构、图片 hash、文本位置 | 仅 Vue 实例存在 |
| 稳定性 | 低（7 个已修复 bug） | 高（结构化数据） |
| 全量采集预估 | 15-20 min | 5-8 min |

#### 注意事项

- 切楼层后需等 **5-10 秒** 让 WebSocket 推完数据（可通过轮询 `Object.keys(websocketDataProp).length` 判断数据就绪）
- 部分 dump 文件中设备名为数字——这是**时序问题**（dump 时 WS 尚未推送名称），非数据缺失
- 如果 `dynType` 未暴露在 `JSON.stringify` 中，可用 `ptPathConf.name` 做中文匹配降级

## 4. 方案三：API 拓扑 + WebSocket 状态分离

### 思路

- **拓扑（静态）**：从 `/api/` 响应一次性获取 6 栋楼的完整设备列表（含楼层/子区归属）
- **状态（动态）**：只通过 WebSocket 获取实时值

### 可行性结论：待验证

- `ems-mirror` 已捕获 106 个资源，含 `/api/` JSON 响应（有 `DevName` 字段）
- **关键问题**：API 是否返回完整设备列表含楼层归属？还是仅当前页？
- 需实际调用一次 API 验证响应结构

### 如果可行

- 拓扑采集：0 秒（一次性 API 调用替代逐层点击）
- 状态采集：仍需导航 + WS 等待，但可用方案二加速
- **全量采集预估：2-3 分钟**

## 5. 补充方案：并行化

Playwright 支持多 browser context，6 栋楼可同时跑：

```javascript
const results = await Promise.all(
  ['1号','2号','3号','4号','5号','6号'].map(b => enumerateBuilding(b))
);
```

**预估效果**：从 15-20 min → 3-5 min（需 6 个 CDP 端口或 6 个独立 Chromium 实例）。

**注意事项**：
- CDP 模式需为每栋楼分配独立端口
- 无 CDP 模式（`--no-headless`）天然支持多 browser context
- 需验证 EMS 是否限制同一账号并发连接

## 7. 改造步骤（方案二）

### 7.1 在 `enumerate.js` 中替换 `extractCards()`

当前 `extractCards()` 位于 `injectHelpers()` 内（约 200 行），替换为：

```javascript
function extractCardsFromVue() {
  const w = document.querySelector('.pi-graphics-configuration-svg-new');
  if (!w || !w.__vue__) return { error: 'no vue instance' };
  
  const d = w.__vue__.$data;
  const rc = d.runConfDataProp || [];
  const ws = d.websocketDataProp || {};
  
  // 等 WS 数据就绪
  if (Object.keys(ws).length < rc.length * 0.5) {
    return { error: 'ws not ready', ready: Object.keys(ws).length, total: rc.length };
  }
  
  const byDev = {};
  for (let i = 0; i < rc.length; i++) {
    const c = rc[i].ptPathConf;
    const devId = c.devId;
    if (!devId) continue;
    
    if (!byDev[devId]) byDev[devId] = { devId, fields: {} };
    byDev[devId].fields[c.name || '__name__'] = {
      idx: i,
      ptId: c.ptId,
      value: ws[String(i)]?.tag?.value,
      valid: ws[String(i)]?.tag?.valid,
      enumDef: c.enumDefine
    };
  }
  return byDev;
}
```

### 7.2 在 `enumerate()` 主循环中替换调用点

```javascript
// 当前：
const cards = await page.evaluate(() => window.__ems.extractCards());

// 改为：
const rawData = await page.evaluate(() => window.__ems.extractCardsFromVue());
const cards = mapVueDataToCards(rawData);
```

node src/enumerate.js --edge --bldg=1号 --append # 采集1号对比结果
